## Guidance

- Sound architecture, modern syntax, English-language comments
- Abstract the essence of problems, apply top-tier programmers' engineering mindset to analyze challenges, and identify optimal solutions and best practices
- Use an `interface` instead of a `type`
- Always run `pnpm lint` before committing
- Use `async/await` instead of `.then()`
- Use `lucide-vue-next` for icons
- Use `shadcn-vue` for components, styling, and theming
- Use `pnpm dlx shadcn-vue@latest add <component>` to add new components if needed
- Use `zod` for schema validation
